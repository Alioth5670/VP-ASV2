你正在修改一个已有的 PyTorch 项目，项目主模型是 **VPAS**，用于 prompt-query 配对输入的变化分割。请基于现有代码实现一个**第一版本的最小闭环方法**，目标是：

2. neck 对 prompt/query patch tokens 做对齐

# 一、方法目标

当前 VPAS 更像 residual-based change segmentation。现在要改成一个**最小版本的新方法**：

* 输入：`prompt_image`, `query_image`
* 输出：直接由 **difference filtering** 得到的 `anomaly-oriented mask`
* 暂时**不要 decoder**
* 暂时**不要 texture branch**
* 核心只验证：

> **difference token 能否在两个可学习 prototype 的监督下，被直接转化为 anomaly-oriented mask**

核心思想：

* difference token 中既有真实异常变化，也有正常环境变化
* 使用两个 prototype 对 difference token 做二分类式 filtering：

  * anomalous-change prototype
  * normal-variation prototype
* 每个 token 的 anomaly probability 直接作为 patch-level mask
* 最终通过 reshape + upsample 得到像素级输出

# 三、整体新流程

请实现下面这个最小流程：

1. `prompt_features = backbone(prompt_image)`
2. `query_features = backbone(query_image)`
3. 对每个层级 `i`：

   * 取 prompt/query 的 patch tokens
   * 用 FARM 做 prompt->query 对齐
   * 得到 `aligned_prompt_tokens`
   * 构造 res
   * 用 prototype filter 计算：

     * `anomaly_prob_tokens`
     * `filtered_difference_tokens`
4. 将 `anomaly_prob_tokens` reshape 成 patch map
5. 上采样到 query 原图大小
6. 多层融合：

   * 第一版优先实现最简单策略
   * 推荐：

     * `single_layer`: 只用最后一层
     * 或 `avg`: 多层直接平均
     * 或 `add`: 多层直接相加

7. 输出最终 `logits`

重点：
**第一版不要再把 filtered difference 喂给 decoder。**
直接把 `anomaly_prob_tokens -> patch map -> upsample -> mask` 作为结果。

# 六、新增 prototype-based discrepancy filter 模块

请新增独立模块，建议文件：

* `models/attention/discrepancy_filter.py`
  或
* `models/neck/discrepancy_filter.py`

## 模块职责

输入：

* `difference_tokens: [B, N, C]`
* 可选 `layer_idx`

输出：

* `anomaly_score: [B, N]`
* `nomaly_score: [B, N]`

## 核心逻辑

模块内部维护两个 prototype：

* anomaly prototype
* normal-variation prototype

对于每个 difference token `d_i`，分别与两个 prototype 计算 cosine similarity：

* `s_a = cosine(d_i, proto_anomaly)`
* `s_n = cosine(d_i, proto_normal)`

再构造 anomaly probability：

* `scores = softmax([s_n, s_a])` 中 anomaly 类对应的概率

最终：

* `scores : [B, N, 2]`

## 要求

* 模块实现清晰、可读、易扩展
* 写清楚张量 shape 注释
* prototype 参数使用 `nn.Parameter` / `nn.ParameterList`
* cosine similarity 实现稳定，注意 normalize

# 七、prototype 设计要求：支持 shared / layerwise

这是当前版本的重点之一。
请将 prototype embedding 设计为**可选的共享 prototype 或分层 prototype**，通过配置控制。

## 配置项

新增例如：

* `prototype_mode: shared | layerwise`

含义：

### `shared`

所有层共享同一组 prototype：

* anomaly prototype
* normal prototype

### `layerwise`

每一层有自己独立的一组 prototype：

* anomaly prototype for layer i
* normal prototype for layer i

---

## discrepancy filter 模块接口要求

支持：

* `difference_tokens: [B, N, C]`
* `layer_idx: Optional[int] = None`

逻辑：

### 当 `prototype_mode == "shared"` 时

* 使用共享 prototype
* `layer_idx` 可忽略

### 当 `prototype_mode == "layerwise"` 时

* 根据 `layer_idx` 选择对应层的 prototype

# 九、监督与损失设计要求

## 损失配置
损失计算修改 `segmentation_loss` 包含以下三项：

* 对 `logits` 计算 `Focal Loss` 得到 `L_Focal`
* 对 `logits[:,:,0]` 预测结果进行 `Dice Loss` 得到 `L_Dice_norm`, label 为 `1-mask`
* 对 `logits[:,:,1]` 预测结果进行 `Dice Loss` 得到 `L_Dice_anorm`, label 为 `mask`


## 总损失建议

总损失形式可为：

* `L = L_Focal + L_Dice_norm  +  L_Dice_anorm`
